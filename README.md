# 🛡️ Deadline Guardian AI
### Autonomous Deadline Rescue System — Google AI Studio Hackathon

> "It doesn't manage your tasks. It prevents you from failing."

---

## 🧠 What Is This?

Deadline Guardian AI is an autonomous AI execution engine that predicts deadline 
failure before it happens — then fixes it automatically.

This is NOT a task manager.
This is NOT a planner.
This is NOT a to-do list.

It is an active rescue system that monitors workload in real time, simulates 
cascade failures, compresses schedules under pressure, and deploys emergency 
protocols when survival is at risk.

---

## 🎯 The Core Problem

Students and developers miss deadlines not because they are lazy — but because 
they cannot see failure coming until it is too late.

Traditional tools show you what to do.
Deadline Guardian shows you what will go wrong — and stops it.

---

## ⚡ Key Features

### 🔴 Deadline Survival Probability
A single master metric (0–100%) calculated in real time from:
- Remaining workload vs time available
- Critical path completion status
- Risk level of active tasks
- Overdue milestone penalties

### 🧠 AI Task Decomposition
Describe any goal. The AI breaks it into real execution steps with 
dependencies, risk levels, and impact analysis. Never generic. 
Always technical. Always dependency-aware.

### ⚡ Cascade Failure Simulator
Simulate skipping or delaying any task. See exactly which downstream 
tasks collapse, how many hours of backlog are created, and what the 
new survival probability becomes — before it happens.

### 🚨 Panic Mode (Signature Feature)
When hours are critically low:
- Removes all low-ROI tasks
- Compresses remaining tasks by 35–40%
- Protects the critical execution path
- Generates a minute-by-minute battle plan
- Shows exactly how much time was saved

### 🔄 Autonomous Replanning
Mark a task as missed. The AI immediately absorbs the delay, reshuffles 
remaining workload, explains the cascade impact, and issues a new 
survival directive.

### 🤖 Tactical Coach AI
A persistent AI strategist active across all pages. References specific 
task names, deadlines, and critical path status. Never gives generic 
motivation. Always gives concrete next actions with time estimates.
